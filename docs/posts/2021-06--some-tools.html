An overview of jupyter notebooks

Jupyter notebooks provide a way to package written material – with possible mathematical markup – together with an interactive environment for executing computer code.

You should follow the installation instructions appropriate for your computer. (I completely acknowledge that this is an at least somewhat complicated task, but I don’t think I have much to add that you won’t find by following this link…!)

Having installed jupyter lab/notebook, you may now use it, as follows. I assume in this discussion that you are sitting at your laptop/desktop computer (i.e. that you aren’t logged into it remotely via ssh or something like that).

Start jupyter from the command-line. I actually find that I prefer the jupyter lab interface to the the plain jupyter notebook interface (and honestly I’m not sure I understand why there are two different interfaces) but you should experiment for yourself.

So, type:

$ jupyter lab

(or

$ jupyter notebook

if you prefer)

There will be some output to the terminal that you can mostly ignore. The immediate effect of running this program is that a page should open in your web browser.

The page that opens will have a URL that looks something like:

http://localhost:8888/lab

(or

http://localhost:8888/tree

)

This URL tells you that there is now a server running on your local machine, and that your web browser is communicating with that server.

By default, jupyter will (likely?) use a python kernel. See below for installing other kernels…

In the web page at this location, you should see in particular an interface where you can interact with jupyter notebooks. You can create a new notebook from the File menu: New -> Notebook. You might (?) be asked to Select Kernel. Immediately following installation of jupyter, you probably only have one choice (python or python 3).

After you select a kernel, you are now editing a file name Untitled.ipynb.

You can end the jupyter notebook session by typing C-c (twice) in the terminal where you started the command, or by menu-driven commands in the web-interface.
